## 0. まず読むファイル
- `CLAUDE.md` … プロジェクト規則・ファイル構造・よくある失敗。

## 1. ハード制約（破ってはいけない）
- 文字コード UTF-8（BOMなし）。Windows/Python 3.13/Streamlit。
- スクレイピングをOCRに置き換えない。
- 既存の関数シグネチャを壊さない（optional引数の追加は可）。
- requirements.txt に無い新ライブラリを足さない。
- スクレイピングは `core/scraper.fetch_robust_html`（Scrapling/Playwright）を使う。`requests`直叩きはbot検知で失敗。

## 2. 統治原則（最重要）
**「検証済みエッジのみ」採用する。** 予測（着順当て）はオッズに織り込まれて+ROIにならない。市場を破れるのは
**消去/危険人気馬(fade)・相手の質・見送り/点数/券種**であって、予測でも資金配分でもない。
- 単勝ROIは全条件・全オッズ帯で控除を抜けない（`verified`）。
- 「どう買うか」だけでROIは上がらない：追い上げ✗/レース選別✗/穴厚✗/エッジ流し✗（全て検証で否定）。
- 中核目標＝過小評価の勝ち馬を拾う（recall@7）。市場を破る側＝危険人気馬の検知・相手の質・的中率/分散管理。

## 3. もう試して却下済み（再提案しない・実装しない）
- **マーチンゲール/追い上げ（モンテカルロ含む）** … 数学的にEV不変・破産確定（3連複10点で-4370万円/破産1185回・実証）。
- **3連複で穴を厚くする** … ROI悪化（77%→64%）。
- **回収率(単勝ROI)で馬を絞る** … 全帯priced-in。
- **俗説5案**（初ブリンカー/距離短縮/お帰り/季節/初ダート）… 正の妙味ゼロ。牝×冬春fadeのみ採用。
- **前走着差0.6/0.8秒・圧勝（単勝）** … 小標本ノイズ/リーク。圧勝馬は単勝過剰人気。
- **ショッカー/逆ショッカー** … 通過順リーク。事前部分はエッジ消滅。
- **巻き返し（次走穴妙味）** … 過剰人気。正は1-3番人気の複勝信頼度UP止まり。
- **展開恩恵・習性脚質を軸/妙味に** … priced-in（残差≈0）。展開position・kyakushitsuは結果リーク源。
- **PCI乖離・5走前理論・補正タイム俗説** … priced-in。
- **血統を予測器(LTR)に足す** … 父×馬場・母父・ニックス・国型まで全てpriced-in（cond2/cond6で否定）。
- **全体の騎手/厩舎成績・全体USM** … priced-in（条件特化のみ有効）。

## 4. 効いている検証済みエッジ（伸ばすならこの上に）
- 🔵補正タイム（直近7走×同馬場の最高）＝本命軸の補強。
- 🔥末脚偏差（人気薄×上がり3F top）＝穴の相手の質。
- 🏠厩舎の当コース勝率≥20% / 騎手の当場・当距離勝率。
- 🌧️道悪×血統：重/不良×1番人気のサンデー瞬発系=危険(消し) / 米国型ダート(シニミニ)=軸補強。
- 危険人気馬：重・不良×1番人気 / 外有利日×内枠の1-3番人気。
- 決着タイプ：ハンデ戦・フルゲート16+=②荒れ寄り / 少頭数8-10=本線(堅)。
- LTR（LightGBM LambdaRank・recall@7=0.936）＝予測の主軸。採用済み特徴=補正T/末脚/厩舎当コース/騎手当場・当距離。

## 5. データの制約（提案の前提）
- 血統(horses)はJV-VANマスタが2023-07凍結→2024-26馬はnetkeibaでバックフィル中。ライブはscraped sire優先で発火。
- win_odds は約37%欠損 → 群間ROI比較は不可（複勝残差＋zで測る）。
- kyakushitsu(結果脚質)・corner通過順は**結果リーク**。特徴量に使わない。
- クッション値/含水率は track_cond に疎（2273日分のみ・2018-07〜）。

## 6. 新案の検証方法（必須プロトコル）
1. `scripts/*_backtest.py` を書く。**1番人気オッズ帯で統制した複勝/勝の残差＋z**で測る（priced-in判定）。
2. リーク厳禁：事前入手可能な変数だけで残差が残るか再確認。通過順/脚質/今走着順を条件に入れない。
3. LTR特徴量は `scripts/auto_feature_search.py`（recall@7・val(2024)とtest(2025+)両改善・マージン・自動デプロイ無）。
4. 変更後 `python tests/smoke.py` が全合格であること。健全性は `python scripts/health_check.py`。
