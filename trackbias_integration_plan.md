## 0. 現状（アプリに既にあるもの）
- **エビデンス表(evidence_list, app.py)**: コース条件/クラス/斤量/開催日数/天候馬場/1番人気オッズ/アノマリー数/先行馬密集度。
  - 「開催日数」≥7日目→🚩外差し有利化、未満→内フラット（粗い）。
- **Vエリア馬場バイアス**: フラット/内2/内4 を開催日数・馬場状態から自動初期化（手動上書き可）。`_V_COL`で内/中/外の有利ゾーンへ。
- **直線の風補正**: Open-Meteo(無料)自動取得＋手動。`wind_effect`で追い風=差し/向かい風=先行・大型。
- **コース諸元**: `get_course_layout`（1角まで距離・直線長・直線/小回り）。
- **finishモデル**: `predict_finish`（4角位置+上がり+適性+総合力+人気）。**展開妙味フラグ**（Vエリア該当×低評価）。
- **jravan.db**: races(tenko, baba_shiba, baba_dirt, mae3f, ato3f, track_code, kyori, surface, hasso_time) /
  results(waku, umaban, corner1-4, time, ato3f, kyakushitsu, chakujun, win_odds, ninki, bataiju)。
  ※**クッション値・含水率・砂厚・ABCコースは未収録**。track_codeは東京芝で全'11'（ABC区別不可）。

## 1. 取り込むべきデータ（入力パラメータ）

### A. JRA公式・馬場情報
- 馬場状態（良/稍重/重/不良） … ✅ baba_shiba/baba_dirt
- **クッション値（芝）** 7以下軟〜12以上硬 … 🟡 JV-VAN取り込み要（JV-Dataのどのrecordか要特定。2020〜）
- **含水率（芝=路盤/ダ=表層、ゴール前・4コーナーの2地点）** … 🟡 JV-VAN取り込み要（地点差＝部分荒れの直接証拠）
- 芝丈・砂厚（標準砂厚9.0cm） … 🟡 JV-VAN/JRA発表（更新頻度低・優先度中）

### B. コース静的データ
- 芝種（野芝/洋芝/オーバーシード。洋芝=札幌・函館でクッション低め） … 🟢 場名マスタを自作
- **ABCDコース区分（仮柵移動）** … 🔴 track_codeに無い。JV-Data別fieldか外部要調査
- 直線長・坂・1角まで距離・大箱/小回り … ✅一部 `get_course_layout`（拡張可）

### C. 馬場整備履歴
- エアレーション/シャタリング（路盤ほぐし→差し有利化）・散水・芝刈り・ハロー掛け … 🔴 構造化データ無し（JRA発表テキスト手動 or スクレイピング。優先度低）

### D. 気象（動的）
- 直線の風向・風速 … ✅ Open-Meteo実装済
- 気温・日照（ダート乾燥速度→含水率急変） … 🟡 Open-Meteoで気温取得可（乾燥ロジックは要設計）

## 2. バイアス評価軸（出力ラベル）

### 内外（コース取り）
- 内有利（インバイアス）: 開幕週・コース替わり直後
- 外有利（アウトバイアス）: 開催後半・雨天で内が傷む
- 外回りイン有利（逆説）: 内荒れで皆が外を回り、最内がぽっかり空く

### 前後（脚質）
- 前有利（先行・前残り）: 高速馬場・直線追い風・ダ湿潤
- 後ろ有利（差し追込）: タフ馬場・エアレーション直後・ダ極乾燥

### 馬場差（時計の出やすさ）
- 走破タイム−基準タイムを秒数化 … 🟢 resultsから算出可（場・距離・馬場別の基準タイムテーブルを自作）

## 3. 芝 / ダート 処理ロジック（パラメータが逆になる）

| 要素 | 芝 | ダート |
|---|---|---|
| クッション値↑(硬) | 前有利・高速 | （芝指標） |
| 含水率↑ | 時計**遅**（脚沈む・スタミナ） | 時計**速**・前有利（砂締まる）※芝と逆 |
| 開催進行 | 内→外へバイアス変化 | ハロー掛けでリセット＝変化出にくい |
| 枠ベース | 開幕内→後半外 | 砂被り回避で**外枠ベース有利** |
| 位置取り | 状況次第 | 先行の価値が芝以上に高い |

→ 実装は `surface` で分岐する補正テーブルにする。

---

## 4. 自動判定ロジック（当日逆算・JV-VANの全レースデータが強み）

- **当日同コースの4角通過順集計** → 1-3番手ばかり=前残り / 10番手以下が複数=差し有利 … 🟢（要：当日前半レースのJV更新取り込み）
- **好走馬の枠番偏り**（上位入線の枠分布） … 🟢
- **上がり3F差**（1位と2位以下の差・複数が速い上がり→差し届く馬場） … 🟢
- **3視点の枠評価**: 通常馬番 / 逆馬番(大外=1) / 外枠率(馬番÷頭数) … 🟢 実装容易
- **コース×バイアス マトリクス** … 🟢 `get_course_layout`拡張
  - 大箱×高速=差し届く / 大箱×時計かかる=先行有利
  - 小回り×高速=内・逃げ先行・イン差し / 小回り×時計=中外・外差し
- **コールドスタート対策**: 当日1R前は静的要因（開催日数・クッション・コース替わり）で暫定 → レース消化で再帰更新

## 5. 予想・買い目への応用（指数化）

- **絶対能力とバイアス恩恵の分離** … 🟢 過去走補正：「バイアスに恵まれた勝ち＝次走危険人気」「バイアスに逆らって好走＝次走穴」を指数化（finish_hist / 強適スコアに接続）
- **不利巻き返し馬の抽出** … 🟢 「外差し馬場で内を通った」「前残り馬場で後方」を次走狙い目リスト化（既存 anom_rushers を拡張）
- **クッション値×血統・走法マッチング** … 🟡 血統DB(build_blood)連携。硬=スピード持続/ストライド、軟=パワー/ピッチ
- **ML（LightGBM等）特徴量** … 🔵 Target Encoding（騎手・種牡馬を勝率変換）、**相互作用特徴量**（クッション×騎手、脚質×砂厚、血統×内側砂厚）。※現アプリは強適スコア方式（非ML）なので別トラック。リサーチ実績: 35年162万件で上位10%勝率15.6%(BL7.45%)・回収率73.7%＝「当たるが儲からない」打破に相互作用が鍵

## 6. アプリ統合ポイント（どこに挿すか）

1. **エビデンス表**に行追加: クッション値 / 含水率 / 馬場差(秒) / 当日逆算バイアス（前残り率・枠偏り）
2. **Vエリア baba** を当日逆算バイアスで自動化（現在は開催日数のみ→実測へ格上げ）
3. **finish / 予測スコア**に「バイアス×脚質×枠」補正（マルチプライヤー）
4. **妙味フラグ**に「バイアス不利巻き返し」候補を追加

---

## 7. 正直な実現可能性 / やらないもの
- 🔴 整備車両の画像解析・騎手コメントNLP・砂付着のリアルタイム検知 → データ源が無く非現実的
- 🔴 公式「重」だが実態「不良」の合成馬場補正 → 当日逆算で近似はできるが厳密再現は不可
- 🔴/🟡 ABCコース区分 → データ源の確定が前提
- ⚠ 各バイアスルールは jravan.db で**必ずバックテスト**してから採用（例: 当日前半の前残り率が後半の前残りを予測するか / クッション値と前-差し決着の相関 / 開催日数と外差し率）。本プロジェクトの「測ってから最適化」を踏襲。

---

## 8. 推奨実装フェーズ
- **Phase 1（取り込み不要・即効）**: 当日逆算バイアス（results集計）＋3視点枠評価 → エビデンス表追加＆Vエリア自動化。先にバックテストで有効性確認。
- **Phase 2**: クッション値・含水率をJV-VANから取り込み → 芝/ダ分岐ロジック → エビデンス表。
- **Phase 3**: バイアス恩恵分離・不利巻き返し抽出 → 予測スコア/妙味へ接続。
- **Phase 4（将来）**: ML特徴量（相互作用・Target Encoding）で別モデル化。
